Team Activity - Android Data Storage

Overview

Before meeting with your team, please review the preparation material so that your time can be as efficient as possible.

For this activity, we will explore some options for storing data in an Android application. To demonstrate this, we will be storing a count of the number of times a user has clicked a button. You are required to implement this functionality using SharedPreferences, and have the opportunity to also do it using a SQLite database for extra credit. If you will be using SQLite in your project, I would strongly recommend completing the extra at this point to get you started.

Instructions

Part I - A simple counter

  1. Create a new, blank Android project.
  2. Create a new activity that has two buttons labeled: "Save Count" and "Advance", and a single text box.
  3. When the activity starts, have it put the number "0" in the text box.
  4. Make the "Advance" button increment the number in the text box by one.

Part II - Data storage

  1. Make the "Save Count" button save the count to the user's SharedPreferences.
  2. Change the activity so that when it loads, it tries to load the saved value from the SharedPreferences. If no value can be found, it should put "0" in the box.
  3. Verify that your saving works by doing the following:
    1. Advance to "4"
    2. Click "Save Count"
    3. Advance to "5"
    4. Close the application, and restart it. The number in the box should be 4.

Part III - Extra Credit

  1. For 25% extra credit. Convert the above logic to use a SQLite database (natively on Android).
