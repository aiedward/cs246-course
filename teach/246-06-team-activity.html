Team Activity: Threads

Instructions

Before meeting with your team for this activity, please read the material in the Team Activity Preparation Material.

Next, meet with your team (e.g., Skype, Google Hangouts) to complete the following activity. This activity is designed to take 1 hour and will help you in your individual assignment for the week.

Assignment

This assignment will be done in parts, as follows:

Part 1 - Simple Multithreading

  1. Write a Java program that simultaneously counts to 100 by odds and even numbers. In other words, you should have two loops that each count to 100 and have these loops run on different threads concurrently.
  2. In order to see the progress, have each thread wait for 100 milliseconds (i.e., "Thread.sleep(100);").

Part 2 - Waiting for threads to finish

  1. Have the program display the message "All finished" when both threads are finished.

Part 3 - Stopping a thread

  1. Add an additional thread that displays the text "running…" every second. This thread should be started concurrently with the other two counting threads. It should be set to continue looping forever (rather than going through a predetermined number of times).
  2. Have the program properly stop the "running…" thread when the two counting threads are both finished.
  3. Please note that the "Thread.stop()" and "Thread.destroy()" methods are deprecated, and are not the proper way to stop the thread.
  4. See this page for more information about why the stop method is deprecated and what you might do: http://docs.oracle.com/javase/7/docs/technotes/guides/concurrency/threadPrimitiveDeprecation.html

Submission

When you have completed the assignment please upload your .java file (or files) to I-Learn. In the comments box, please specify which of the following were completed:

  • Every member of the team was present and contributed.
  • Completed part 1
  • Completed part 2
  • Completed part 3
