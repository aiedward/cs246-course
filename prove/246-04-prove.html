Version Control Assignment

Overview

This exercise will give you a chance to see how committed you are to using version control and finding things among source that has been committed to a version control system.

In particular, for this exercise we will be using Git, signing up for an account at GitHub, committing files to it, pulling files from it, and looking through the history of a Git repository.

Use the internet and other students as a resource. The official git tutorial is at: http://git-scm.com/docs/gittutorial

Instructions

The following steps will guide you through this activity. Each one is pass/fail.

Part I – Collaboration – 5%

  1. (5%) Collaborate with others on this assignment.

Part II – Committing to a repository – 70%

  1. (10%) Download and install Git on your laptop: http://git-scm.com/downloads
    • You are welcome to use a GUI client if you like, but as these will be different across platforms, examples and class discussion will focus exclusively on the command line version.
    • If you are using Windows, after you install git, you will see "git-bash" this is the command line terminal you should use (rather than the built-in Windows one). It is a light-weight Linux-style command line terminal, that helps to run the git commands.
  2. (10%) Sign up for a free account at GitHub.com.
    • Keep in mind that what you commit to your public repositories will be visible to rest of the world.
  3. (5%) Create a new repository at GitHub for use in this class (e.g., cs246)
    • For the dropdown, “add .gitignore”, select Java. This will add default ignore options for things like *.class and *.jar files.
  4. (5%) Create a file, HelloWorld.java that simply displays “Hello World” to the console.
  5. (10%) Commit your HelloWorld.java file to a local repository and push it to your GitHub repository.
  6. (10%) Change your HelloWorld.java program to display “Hello GitHub World”, and push this change to your GitHub repository.
  7. (5%) Create a branch on your local machine named, “test”.
  8. (5%) Change your HelloWorld.java program to display the line “Testing…” after the previous line (“Hello GitHub World”), and push this to a branch at GitHub named, “test”.
  9. (5%) Go back to your master branch, and remove the word “World” from the output, so that it now displays, “Hello GitHub”. Push this change to your GitHub repository (on the master branch).
  10. (5%) On your local machine, merge the changes from your test branch back into your master branch (the result should be a program that displays two lines “Hello GitHub” and “Testing…”). Push this to GitHub on your “master” branch.

Part III – Using the history of a repository – 25%

  1. (2%) Clone the following repository: https://github.com/sburton42/cs246-committedness.git
  2. (3%) Identify the most recent commit message that has been made to the repository.
  3. (3%) Identify the file that was changed in the most recent commit.
  4. (3%) Identify the message of the first commit to this repository.
  5. (3%) Identify the file that was added in the first commit.
  6. (3%) What time of day (hh:mm:ss) was the commit with the message “adding three great classes” made?
  7. (2%) Identify the commit message when the file “file3.txt” was added.
  8. (2%) The class Broken has a compile error. Identify the commit message of the revision in which it was broken.
  9. (2%) The class Fixed had a compile error at one point, but then was subsequently fixed. Identify the commit message of the revision in which it was fixed.
  10. (2%) The class Deleted was in the repository at one point, but has subsequently been deleted. Identify the commit message of the revision in which it was removed.

Submission Instructions

To “submit” your assignment, answer the quiz questions on I-Learn. You are required to collaborate with others on this assignment, however, do not just enter in the answer (e.g. a certain commit message) found by another student. You should be able to pull up the message on your own screen if you answer the question. Thus, you should not just share the “answer”, but rather, share the process to find the answer.
